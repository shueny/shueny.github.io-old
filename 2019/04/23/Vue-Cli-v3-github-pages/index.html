Vue Cli v3.0安裝/建置 + Bootstrap + SCSS + Github Page

نُشر في 2019-04-23 في Vue.js

前置作業

Node 版本要求

Vue CLI 需要 Node.js 8.9 或更高版本 (推荐 8.11.0+)。
可以使用 nvm 或 nvm-windows在同一台電腦中管理多個 Node 版本。

安裝

安裝完後可以用以下命令檢查版本是否正確

https://cli.vuejs.org/zh/guide/installation.html

設定步驟

Step1. 建立 Vue.js 專案項目

vue create side-projects

  • 建立一個自訂名稱的專案項目 ( 此為 ‘side-projects’ )

Step2. 專案建置設定

Step3. 進入專案資料夾中並測試

檔案結構

side-projects/        (專案目錄)
│
├── dist/           (使用 npm run build 構建完成的檔案)
│
├── node_modules/   (依賴的模組)
│
├── public/         (公共資源檔案，不做任何編譯壓縮處理，使用絕對路徑指定的檔案)
│   │
│   ├── favicon.ico (icon 圖片)
│   └── index.html  (網站入口)
│
├── src/            (Vue 與前端的原始檔案)
│   │
│   ├── assets/     (公共資源檔案，與 public 不同的是，assets 會經由 webpacd 構建處理，使用相對路徑指定的檔案，如 LOGO 圖片)
│   ├── components/ (Vue 公共組件)
│   ├── views/      (Vue Router 頁面組件)
│   ├── App.vue     (項目入口)
│   ├── main.js     (項目的核心)
│   ├── router.js   (路由設定)
│   └── store.js    (Vuex 狀態管理)
│
└── package.json    (相當於對整個項目的解釋說明文件)

Step4. 安裝bootstrap

npm install bootstrap jquery popper.js

Step5. import Bootstrap

在 src/main.js 中匯入 Bootstrap 的 js 與 css

Step6. github 新增 repo 並與專案相連接

Create a New Repo

使用 push an existing repository from the command line

Step7. 新增 vue.config.js 及設定

在 根目錄下 新增 vue.config.js 這個檔案

(因 Vue CLI v3 後，已 webpack 的設定檔案完全隱藏，在目錄中完全找不到 webpack 的 config 檔。所以，我們需要自行調整 webpack 的設定時，需另外在專案的根目錄建立 vue.config.js 來設定這些細節。若無新增此檔案，則使用預設值)

Step8. 修改 .gitignore

打開 .gitignore 檔案，並把 /dist 移除 或 設為註解

Step9. 把專案 Build 起來

npm run build

Step10. 先把 /dist, gitignore and vue.config 之外的檔案先做 commit

IMPORTANT!!

Before you run the next command make sure you don’t commit the .gitignore and vue.config.js.

Step11. add dist and commit

Step12. 用 git subtree 推送到 github 上

git subtree push -- prefix dist origin gh-pages

Step13. 設定 github page

檔案更新

若之後檔案有更新要 push 上去，在 github page 部分則重複 Step9 ~ Step12
